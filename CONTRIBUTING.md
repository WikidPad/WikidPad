# Contributing to WikidPad

Thank you for your interest in contributing to WikidPad.

## Report bugs
Please use GitHub Issues to report bugs:
- Provide a short, descriptive title.
- Describe what you expected to happen and what actually happened.
- Give clear, minimal steps to reproduce the problem.
- Include your WikidPad version, operating system, and any relevant log output or screenshots.
- Link related issues or pull requests if relevant.

## Contributing code or docs
When you submit a pull request:
- Keep changes small and focused (one issue/feature per PR).
- Keep code simple and easy to understand — prefer clarity over cleverness.
- Stick to the existing coding conventions and style used in the repository.
- Reference the related issue in the PR description.
- Explain the purpose and high-level approach in the PR description.
- Add or update tests where applicable and verify that existing functionality is not broken.
- If you change user-facing behavior, update documentation or help text.

## Commit messages & PRs
- Use descriptive commit messages. One-line summary plus short body when helpful.
- Squash or tidy up WIP commits before the final merge if asked by maintainers.

We appreciate your help keeping WikidPad simple, readable, and useful.
